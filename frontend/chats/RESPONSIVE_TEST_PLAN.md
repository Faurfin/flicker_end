# План проверки адаптивности и кросс-браузерности

## 1. Устройства и разрешения

| Устройство / ширина | Проверка |
|---------------------|----------|
| iPhone SE 320px | Список чатов и окно чата без горизонтального скролла, кнопки не менее 44×44px по зоне нажатия |
| 375px, 414px | То же + безопасные зоны (safe-area) на полноэкранных экранах |
| Планшет 768–1024px | Список чатов ~360–45vw, сайдбар скрыт, переход список ↔ чат |
| Ноутбук 1280px | Сайдбар + список 660px + окно чата, всё видно |
| Full HD 1920px, 2560px | Окно чата ограничено max-width, нет растягивания на весь экран |

## 2. Браузеры

- **Chrome/Edge 90+**: полная поддержка (dvh, gap, backdrop-filter, scrollbar-color).
- **Firefox 88+**: скроллбары через `scrollbar-width` / `scrollbar-color`; `backdrop-filter` при необходимости проверить с включённым флагом или ожидать поддержку.
- **Safari 14+ (macOS/iOS)**: safe-area, `-webkit-backdrop-filter`, `100dvh`; при `backdrop-filter` недоступен применяется fallback без размытия.
- **Samsung Internet**: аналогично Chrome, проверить на реальном устройстве.

## 3. Ориентация и клавиатура

- **Портрет → ландшафт**: при повороте экрана интерфейс перестраивается, состояние (открытый чат, скролл) не теряется.
- **Виртуальная клавиатура (мобильный)**: при фокусе в поле ввода используется `100dvh`/`min-height`, панель ввода и контент остаются доступными, нижняя safe-area учтена.

## 4. Анимации и доступность

- **prefers-reduced-motion: reduce**: при включённой настройке на body вешается класс `reduce-motion`, анимации и длинные переходы отключаются в `responsive.css`.
- **Горячие клавиши**: Ctrl+Enter (или Cmd+Enter на macOS) — отправить сообщение при фокусе в поле ввода; Ctrl+K / Cmd+K — фокус в поле поиска по чатам.
- **Фокус**: у кнопок и полей при навигации с клавиатуры виден `:focus-visible` (обводка 2px).

## 5. Плотность пикселей (Retina/HiDPI)

- При 2dppx и выше элементы интерфейса остаются чёткими; при необходимости проверять иконки и крупные изображения в чате.

## 6. Критерии успеха (чек-лист)

- [ ] Нет горизонтального скролла на ширине от 320px до 900px.
- [ ] Все кликабельные элементы имеют эффективную зону нажатия не менее 44×44px.
- [ ] На iPhone с «чёлкой» контент не заходит под вырез и home indicator (учёт safe-area).
- [ ] При уменьшении движений анимации отключаются (проверка через настройки ОС или DevTools).
- [ ] Ctrl+Enter отправляет сообщение, Ctrl+K переводит фокус в поиск.
- [ ] В Chrome, Firefox, Safari и Edge интерфейс отображается без критичных визуальных отличий.
- [ ] Масштаб страницы 200% не ломает раскладку и не обрезает контент.

## 7. Файлы, затронутые изменениями

- `chats.html` — viewport, meta theme-color, apple-mobile-web-app, подключение `responsive.css`.
- `css/responsive.css` — новый файл: breakpoints, safe-area, touch-таргеты, `prefers-reduced-motion`, скроллбары (WebKit + Firefox), `:focus-visible`, backdrop-filter fallback.
- `css/base.css` — `-webkit-tap-highlight-color` на `html`, `box-sizing` для контейнеров.
- `js/main.js` — класс `reduce-motion` по `prefers-reduced-motion`, горячие клавиши Ctrl+Enter и Ctrl+K.
